## FSH Workflow Instructions

1. Before creating or editing any FSH profile or instance, read C:\FHIR\fhir-context.md first to resolve the full inheritance chain and all aliases.

2. When creating a profile: after the user approves it, create a FSH example instance that tests as many of the profile's constraints and changes as possible. The example must pass validation. If new slices are defined, include entries that exercise them. If you need specific codes, systems, or values to populate the example correctly, ask the user before creating it.

3. After completing any task (profile, instance, or other resource), double-check your work before reporting it as done.
